# 温言助手（SoftTalk）
## 一、产品核心信息
产品名称：温言助手（SoftTalk）

一句话 Slogan：打字不踩雷，沟通更温和 ——AI 实时校准你的文字语气

### 用户痛点（真实可感知，无臆想）
1. 职场沟通：怕打字语气太冲 / 生硬，得罪同事、领导，尤其跨部门对接、向上汇报时；
2. 亲密关系：家人 / 情侣间因文字无情绪，单字回复、直白话引发矛盾，线下解释成本高；
3. 日常社交：社恐 / 不善表达者，打字不知如何把握语气，既怕太随意又怕太拘谨。

**核心痛点**：文字沟通缺乏情绪传递，易产生误解，现有工具无「实时情绪检测 + 场景化改写」的轻量化解决方案。

### 解决方案 & 核心 AI 技术
#### 核心功能（极简 2 大功能，小而美）
1. 实时情绪风险检测：悬浮窗挂载于输入框（微信 / 钉钉 / 短信等），边打字边识别文字的情绪风险，实时提示「太冲 / 冷漠 / 过度随意 / 不正式」等问题；
2. 一键场景化软化改写：点击提示即可生成 3 类语气版本，一键替换输入框内容，不改变原意仅优化语气。

#### 核心 AI 技术
基于LLM 大模型（GPT-4/Claude/DeepSeek 等） 搭建轻量规则引擎 + 小模型微调，实现「文本实时采集 - 情绪特征提取 - 风险等级判定 - 场景化文本改写」全流程，无复杂视觉 / 语音技术，聚焦 LLM 自然语言处理能力。

#### AI 参与度（全链路 AI 辅助，可附 Prompt 截图）
1. 创意发散：通过大模型挖掘文字沟通的真实痛点，筛选「情绪脱敏」核心方向，排除非刚需功能；
2. 技术框架：让大模型生成「悬浮窗 + LLM 情绪判断」的代码框架（Android/iOS 端），输出本地规则引擎的核心代码；
3. 内容构建：由 AI 生成「职场 / 朋友 / 家庭」三大场景的语气改写模板库，覆盖超 200 + 高频沟通句式；
4. 视觉设计：用 AI 生成产品 UI 概念图（悬浮窗、操作界面），无需人工手绘；
5. 文档撰写：让大模型辅助完成产品概念单页、商业计划书、路演 PPT 大纲。

## 二、可视化原型规划（适配交付物要求）
**必备**：AI 生成产品概念图（UI 图）
1. 手机聊天框旁的悬浮窗样式（显示情绪风险提示「这句话有点冲」）；
2. 点击悬浮窗后弹出的 3 个场景版本按钮（职场礼貌版 / 朋友温和版 / 家庭不伤感情版）；
3. 点击版本后输入框文字自动替换的效果展示。

**加分项**：简易代码 Demo（低成本可实现）
基于大模型生成的 Python/Android 基础代码，实现黑框框文字对话版 Demo：
1. 输入任意待改写文字，黑框实时输出「情绪风险判定结果」；
2. 输入指令「生成职场版」/「生成家庭版」，黑框快速输出改写后的内容；
3. 录制 30 秒操作屏录，展示核心功能跑通过程，满足路演可视化要求。

## 三、导师招募画像（精准匹配项目需求，反向招募）
结合项目技术难点与产品设计需求，明确 3 类核心导师画像，精准指向行业技术 / 产品专家，符合征集令「用项目吸引导师做合伙人」的目标：
1. 移动端开发工程师（Android/iOS）
    需求原因：项目核心载体为手机悬浮窗工具，需解决「输入框文本监听、悬浮窗权限适配、各 APP 兼容（微信 / 钉钉等）」等移动端技术问题，新手难以快速攻克权限适配与兼容难题。
2. NLP 自然语言处理算法工程师
    需求原因：需优化 LLM 小模型的「情绪判断精准度」与「改写语气自然度」，搭建轻量化本地规则引擎，让 AI 判断更贴合日常沟通场景，避免改写内容生硬、脱离原意。
3. 互联网产品经理（工具类产品方向）
    需求原因：需优化产品交互设计，让悬浮窗操作更轻量化（如一键开关、智能隐藏），同时梳理产品迭代路径（从免费基础版到付费场景版），设计符合用户使用习惯的产品流程。

## 四、项目核心亮点（适配路演吸睛，反向招募导师 / 投资人）
1. 赛道精准：聚焦 Track B 效率的革命，切中「文字沟通效率」细分赛道，无同质化竞争，市面暂无同类完整解决方案；
2. AI First 贯穿：全流程无人工裸写，从创意到代码、设计、文档均由 AI 辅助，完全契合征集令铁律；
3. 落地难度低：无需复杂硬件 / 高端技术，仅靠 LLM 大模型 + 移动端基础开发，可快速产出 Demo，满足 VC 初审的概念验证要求；
4. 商业潜力大：痛点高频、用户群体广（职场人、学生、普通社交者），可通过「免费基础版 + 付费场景高级版」实现变现，后期可拓展为办公插件，适配钉钉 / 企微等平台。
